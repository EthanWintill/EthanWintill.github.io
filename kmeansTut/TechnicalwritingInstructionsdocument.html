How to Implement K-Means Clustering From Scratch In Jupyter Notebook

Disclaimer: While no programming knowledge is necessary to follow this tutorial, it will be very difficult to understand what’s going on without a basic understanding of python.

Introduction

Given large unlabeled datasets, it can be useful to find ‘clusters’ of the data. That is, where the data seems to pool together. K-Means is one of the most popular clustering algorithms in machine learning. In this tutorial, I’ll be walking you through how to write it from scratch in Jupyter Notebook. This tutorial should take around 10 minutes, plus however long it takes you to install the required packages. Which could be between 6 and 120 minutes depending on how familiar you are with python.

Note: If you’re simply looking to use K-Means, it is much easier to use a library. Sklearn has it here.

But First -- More Background

An important foundation to understanding K-Means but also clustering algorithms in general is the idea of featurespace. Featurespace is a very useful spatial interpretation of a dataset. One can imagine the columns of a dataset are different dimensions in space, and each row is a point in that space. So for example, the following table:

How to Find Clusters Using K-Means

Note: The line

sns.scatterplot(df['col1'],df['col2'], hue=df['cluster'])

Appears many times in this tutorial. This is to graph the featurespace of the data. This is not necessary for KMeans, instead it’s for you to check what you have to make sure it looks similar. However, this will not work for feature space with more than two dimensions.

  1. Open Jupyter notebook and import pandas, numpy, and sns.

1.1 If you don’t have these libraries or don’t have Jupyter Notebook, they are all included in Anaconda.

  1. Load the data into a dataframe. Here I used a handmade .csv, but anything will work as long as it can be put into a dataframe. For more information on .csv files go here.

df = pd.read_csv('/Users/mymac/Downloads/DataMining/Proj3/input4.csv', delimiter='\t')

2.1 Here’s the featurespace of the data I’m using.

sns.scatterplot(df['col1'],df['col2'])

  1. First choose the number of clusters, k, that we want to find. For this example I choose k=4.

k = 4

  1. Store the length of the dataframe.

c1_index = len(df.index) #centroids will be appended to end of dataframe, this stores their index

  1. Add a column to df to store each point’s cluster. Because the clusters haven’t been computed yet, these assignments can be random.

col3 = [0] * (c1_index) #create an empty column to store clusters

for i in range(c1_index):

   col3[i] = (i%k)+1 #assign clusters to columns in order

df['cluster'] = col3 #append new column to data

5.1 Here’s the same featurespace with each point colored based on what cluster it belongs to.

sns.scatterplot(df['col1'],df['col2'], hue=df['cluster'])

6. Next, calculate the location of each cluster. In K-Means, the location of a cluster is the mean value of all the points in the cluster. Luckily Pandas has a .mean() function built in.

More Detail: These aren’t the literal locations of each cluster as in the center of the four blobs of dots shown above. Instead, this is the algorithms first (very bad) guess at where the clusters are. As the KMeans progresses, these guesses will get better until they are exactly what visually appears to be the center of each cluster.

7. Append each cluster location to the end of the dataframe.

for i in range(k): #loop through clusters

   cluster = df.loc[df['cluster'] == i+1] #get all the instances in cluster

   df.loc[c1_index+i] = [cluster['col1'].mean(), cluster['col2'].mean(), 1+k+i] #append their mean column values to data this is the centroid

7.1 Here they are graphed, the darker dots are cluster locations:

sns.scatterplot(df['col1'],df['col2'], hue=df['cluster'])

8.  Then, recompute each point’s cluster. As stated in step 4, this is simply the cluster it’s closest to. To find this we will simply calculate the distance to each cluster and assign it to whichever one is the smallest. The last line displays the featurespace.

for i in range(c1_index):

       fclust = df.iloc[i:i+1,2:3].values[0][0]

       cluster_distances = [0] * k

       for j in range(k):

           cluster_distances[j] = np.linalg.norm(df.iloc[i:i+1,0:2].values[0] - df.iloc[c1_index+j:c1_index+j+1,0:2].values[0])

     

       nclust = cluster_distances.index(min(cluster_distances))+1

       if(nclust != fclust):

           cluster_changed = True

       df.iloc[i:i+1,2:3].values[0][0] = nclust

sns.scatterplot(df['col1'],df['col2'], hue=df['cluster'])

9. Finally, we must repeat steps 7 and 8 until the clusters stop changing. Here’s the order of updated featurespace which each repeat of step 6 and 7.

Conclusion

Now that KMeans has been implemented, hopefully you can now effectively cluster numerical data. And more importantly, hopefully you have a better understanding of how KMeans works. If you are getting errors or unideal results, here’s some troubleshooting tips.

  • If the error mentions ‘pd’, ‘np’, or ‘sns’ not being recognized, it means they have not been installed properly. Try uninstalling and reinstalling.
  • If the error message is different, make sure you have copied the code down exactly as it appears in this tutorial.
  • If there’s no error but the graph looks wrong, make sure your featurespace is 2D. The algorithm is likely still working, but sns can only graph 2 dimensional featurespace.
